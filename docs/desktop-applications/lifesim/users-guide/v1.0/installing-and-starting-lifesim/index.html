Installing and Starting LifeSim

This chapter describes the recommended computer system requirements for installing and running LifeSim. The software includes a graphical user interface (GUI), designed to increase the usability and efficiency of the software. The LifeSim GUI provides for file management, data entry and editing, and viewing output results.

Requirements​

  • Installation – about 180MB (megabytes) of hard disk space

  • Memory – 1GB (gigabytes) RAM (random access memory); recommended 4GB RAM

  • Operating System – Microsoft Windows XP, Windows 7, Windows 8/8.1/Windows 10

  • Microsoft .NET Framework Version 4.0

Installation​

  • Install LifeSim using the self-extracting setup package file downloaded from the Risk Management Center (RMC) website ().

  • The installer (with administrative privileges) will run through a series of steps regarding terms and conditions, install location (e.g., C:\Program Files), and shortcut icons.

  • Install the LifeSim example dataset from the RMC website ().

Starting LifeSim​

When starting the LifeSim, double-click the LifeSim icon (shortcut) on the desktop. The main window of the LifeSim will appear (Loading...). If the desktop shortcut icon is not available, the user can start LifeSim from the Start menu, click All Programs, click RMC, and click LifeSim.

Creating a Study​

A study in LifeSim is a combination of the data, models, or events required to analyze a specific geographical area. The study represents all the input data and simulation output required to adequately answer the question of estimating consequences from a hazard. There are two ways to create a study.

Loading...

To create a LifeSim study:

  1. From the LifeSim main window (Loading...), from the File menu (Loading...), click Create New (Loading...). Another way is from the Study Tree, right-click New Study, from the shortcut menu, click Create New. Either way, the Save As browser will open ( Loading...).
Loading...

  1. From the Save As browser, navigate to the location (directory) where the LifeSim study will be created. Enter a name (required) in the File name box.

  2. Click Save, the Save As browser (Loading...) will close. The LifeSim main window (Loading...) will now have the name of the study in the Study Tree and the Title Bar.

Loading...
  1. A directory with the study name and associated files will be created for the study.

Opening a Study​

It is important to note that at this time LifeSim Version 2.0 cannot open or update LifeSim Version 1.0 studies. To open an existing LifeSim study:

  1. From the LifeSim main window (Loading...), from the File menu (Loading...), click Open Existing (Loading...). Another way is from the Study Tree, right-click New Study, from the shortcut menu, click Open Existing. Either way, an Open browser will open ( Loading...).
Loading...

  1. Browse to the location of the LifeSim study file (*.fia). Click on the *.fia file and the File name box will now contain the filename (e.g., LifeSim_Ex_Project_v20.fia).

  2. Click Open, the Open browser (Loading...) will close and the LifeSim main window ( Loading...) will now display the selected study in the Study Tree and the study name will display in the Title Bar.

Study Directories and Files​

Directories and files are automatically created by LifeSim to store information. As data is entered, subsequent files and subdirectories are created. A description of the created directories is provided below:

  • LifeSim Example Model: Study folder that is created when a user creates a new study (Import from HEC-RAS)

  • Deming Fake Dam Breach.fia: File created by LifeSim Version 2.0 when a study is created. Contains the name and description of the study, map properties, and references to the LifeSim study elements. The *.fia file also holds the following study data:

    • Structure Inventory: Contains all of the information about the imported and defined structure inventory, damage categories and structure occupancy types.

    • Emergency Planning Zones (EPZ) Data: Contains all of the information about the imported and defined EPZ data.

    • Road Network: Contains all of the information about the imported and defined road network.

    • Destination Data: Contains the created destination data.

    • Alternative Data: The created alternative is saved as an XML file.

    • Simulations: Contains all of the created and computed simulation data.

  • Hydraulic_Data: Sub-directory created by LifeSim when hydraulic data is imported. Contains files that define data items for imported hydraulic data. Also, sub-directories are created when hydraulic data is defined and are dependent on the hydraulic data import option utilized.

  • Agricultural_Data: Sub-directory created by LifeSim when agriculture data is imported. Contains the created agriculture data geotiff.
