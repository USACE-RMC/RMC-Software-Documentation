# Viewing LifeSim Results

When a simulation has been created for an LifeSim study the simulation should be computed and the results reviewed. During a simulation compute,
summary results for each iteration are written out to a database, the user can then generate detailed iteration results to view agent level simulation
 data such as animations and evacuation timing. Agent level refers to viewing results down to the vehicle or evacuating group level. The detailed
iteration results are computed per iteration to save both storage space and computation time. Results include summary tabulated results by iteration,
summary box and whisker plots, plot by iteration and plot by uncertainty. In addition, detailed output can be generated by iteration allowing the user
 to animate the simulation in the map window and view detailed iteration plot results.

## Simulation Results Options

LifeSim Version 2.0 provides users with the ability to view summary results for all simulations or results for individual simulations. To view the
simulation summary results for all simulations, from the LifeSim main window (<FigReference figKey="figure-189" />), from the Study tab, from the
Study Tree, right-click Simulations (<FigReference figKey="figure-189" />), and click Simulation Result Plots or Simulation Result Table from the
shortcut menu. Alternatively, from the Study menu, point to Simulations and click View Results Plot or View Results Table from the sub-menu (
{"\n"}<FigReference figKey="figure-190" />). [Open Summary Results](./14-viewing-lifesim-results.mdx#open-summary-results) provides an overview of the
 summary results for all simulations.

NOTE: All results plot windows contain the same general plotting tools (located on the right side of the plot window) used for customizing plots.
Review [Customizing Plots and Plotting Tools](./04-lifesim-interface.mdx#customizing-plots-and-plotting-tools) for instructions on using the general
plotting tools.

## Open Summary Results

Users can view the simulation summary results in a table or in a plot window. To view the summary results, from the LifeSim main window (
{"\n"}<FigReference figKey="figure-189" />), from the <strong>Study</strong> tab, from the <strong>Study Tree</strong>, right-click
<strong>Simulations</strong> (<FigReference figKey="figure-189" />), and click <strong>Simulation Result Plots</strong> or
<strong>Simulation</strong> <strong>Result Table</strong> from the shortcut menu. Alternatively, from the <strong>Study</strong> menu, point to
<strong>Simulations</strong> and click <strong>View</strong> <strong>Results</strong> <strong>Plot</strong> or <strong>View</strong>
<strong>Results</strong> <strong>Table</strong> from the sub-menu (<FigReference figKey="figure-190" />).

#### Iteration Plots Tab

In addition to the <strong>Simulation Results By Iteration</strong> and <strong>Filter</strong> panels (<FigReference figKey="figure-201" />), the
<strong>Iteration Plots</strong> tab contains a panel at the bottom of the plot window. The panel provides a <strong>Simulation</strong>,

<strong>Generate</strong> <strong>Detail</strong> button (<FigReference figKey="figure-201" />). The purpose of the panel is to allow users to set
the simulation, alternative, and imminent hazard to display in the plot. Then from the combined selection users can visualize each iteration result in
 a scatter plot to help the user identify which iterations calculated outlying values (e.g., <em><strong>Iteration Number: 90</strong></em> in
{"\n"}<FigReference figKey="figure-201" />). These outlying values, or any iteration of interest, can then be selected to generate detailed output for
 that specific iteration, by clicking the
{"\n"}<FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-ea6edc69.png" /> <strong>Generate Detail</strong>
button (<FigReference figKey="figure-201" />).

<Figure
  figKey="figure-204"
  src="figures/desktop-applications/lifesim/users-guide/v1.0/figures/figure204.png"
  alt="Clear Existing Message Dialog Box"
  caption="Clear Existing Message Dialog Box"

If detailed output is generated, LifeSim re-runs the specified iteration with the same random number sequence but this time much more information is
saved. The detailed output generated contains information down to the evacuating group level regarding group size, location, and timing information
during the simulation. The detailed output generated includes a database containing evacuation information for groups and their evacuation paths,
structure inventory results shapefile, and a road network results shapefile. There is no limit to the number of iterations that LifeSim can generate a
 detailed output for, however physical memory could become an issue at some point. Detailed outputs cannot be generated concurrently.

The <strong>Re-Generate Detailed Output</strong>
{"\n"}<FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-6ba696c0.png" /> option re-generates the specified
 iteration's detailed output and is only available after a detailed output has been generated for a selected iteration. The <strong>Re-Generate
Detailed Output</strong> option is best used when the simulation has been altered and a detailed output had already been created for specific
iteration(s) and would need to be re-generated for the new simulation.

Refer to [View Detailed Output Results](./14-viewing-lifesim-results.mdx#view-detailed-output-results), [Detailed Output
Results](./14-viewing-lifesim-results.mdx#detailed-output-results) and [View Result Map in the Map
Window](./14-viewing-lifesim-results.mdx#view-result-map-in-the-map-window) for more information on viewing the results of the generated detailed
output for iterations.

The functionality of the table is the same as the summary results table ([Simulation Results
Table](./14-viewing-lifesim-results.mdx#simulation-results-table)). The detailed output table contains information about each evacuating group’s

mobilization information, and if they were caught evacuating.

The road and structure summary or detailed output can be viewed for an iteration (e.g., <em><strong>Iteration 97 Road Details</strong></em> in

supplementary investigation.
