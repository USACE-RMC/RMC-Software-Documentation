# Appendix B - Map Preferences

To help the user visualize the study and results, LifeSim has the capability to display map layers that represent the features of a study. The map
layers displayed in the map window are also listed in the <strong>Map Layers</strong> tab (<FigReference figKey="figure-228" />). Map layers provide
a geographical reference for the study area. The LifeSim GUI (<FigReference figKey="figure-228" />) describes the formats of the map layers, and
provides commands that allow the user to configure, manage, and edit map layers.

<Figure
  figKey="figure-231"
  src="figures/desktop-applications/lifesim/users-guide/v1.0/figures/figure231.png"
  alt="LifeSim Map Window – Animation Playback Toolbar – Evacuation and Hydraulic Animation – Example"
  caption="LifeSim Map Window – Animation Playback Toolbar – Evacuation and Hydraulic Animation – Example"
/>

Map layers are optional and not necessary for computing and reviewing results ([Viewing LifeSim
Results](./15-viewing-lifesim-results.mdx#viewing-lifesim-results). However, to view the results of simulation animations ([View Result Map In the Map
Window](#view-result-map-in-the-map-window)) the map window is useful for helping to facilitate the review of simulation results and discussion with
emergency managers. Furthermore, viewing map layers in the map window is a useful way to examine the validity of imported datasets. For example, a
road network with missing connectivity or incorrect directions on one-way roads can lead to erroneous results. Therefore, editing the road network
dataset can provide a reliable foundation for traffic simulation ([Appendix
C](./19-appendix-editing-road-network-data-for-use-in-lifesim.mdx#editing-road-network-data-for-use-in-lifesim) provides an example of editing an imported
road network dataset in the map window).

## Map Window

The map window (<FigReference figKey="figure-228" />) provides a way to graphically display LifeSim components. The rendering engine takes advantage
of the computers graphics processing using Open Graphics Library - OpenGL () to draw the map layers. All geographic information system (GIS) data is
re-projected on the fly using GDAL (Geospatial Data Abstraction Library) ().

### Map Window Toolbars

There are three toolbars located in the LifeSim main window <strong>Map Window Toolbox</strong> (<FigReference figKey="figure-228" />). The first is
the <strong>General Toolbar</strong>, second is the <strong>Selectable Layers</strong> <strong>Toolbar</strong>, and third is the <strong>Editor
Toolbar</strong>. Details about each toolbar are detailed in the following sections.

#### General Toolbar

The General Toolbar includes the Select Tool (B), Pan Tool (P), Zoom In Tool (Z), Zoom to full extents, Add Data, Query Features Tool (Q), and Measure
Distance Tool (M). These tools change the appearance of the cursor, as well as the functionality of the mouse cursor in the map window.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-f5089b1a.png" /><strong> Select Tool (B):
</strong>The <strong>Select</strong> tool allows the user to select elements displayed in the map window that are currently selected in the
<strong>Selectable Layers</strong>. More details on <strong>Selectable Layers</strong> can be found in the next section. To select features, click on
a feature or click, hold, and drag a rectangle around features to select them in the map window. Additionally, the <strong>Select Tool</strong> allows
 users to access shortcut menus which can be used to customize features when in editing mode.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-c6751277.png" /><strong> Pan Tool (P): </strong>The
<strong>Pan</strong> tool allows the user to pan the map window; click the <strong>Pan</strong> tool and click, hold and move the mouse around the map
 window. Also, if the mouse has a wheel, then the user can pan the map window by using the wheel (click and hold the wheel to pan).

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-6cd2df7a.png" /><strong> Zoom In Tool (Z):
</strong>The <strong>Zoom In</strong> tool allows the user to zoom in to areas of the map window. To zoom in, hold the mouse button down and outline
the area of interest to enlarge. Also, if the mouse has a wheel then the user can zoom in or out using the wheel (wheel up zooms in and wheel down
zooms out).

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-2934ef6e.png" /><strong> Fixed Zoom Out Tool (O):
</strong>The <strong>Fixed Zoom Out </strong>tool<strong> </strong>zooms out a fixed distance. Click the <strong>Fixed Zoom Out</strong> tool once and
 the LifeSim map window view will zoom out a fixed distance. Double-click on the <strong>Fixed Zoom Out</strong> tool and the map window view will
zoom out twice the fixed distance.
Additionally, the user can utilize the shortcut menu option, right-click anywhere in the map window, to zoom in or out a fixed distance.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-ddd93902.png" /><strong> Zoom to full extents
Tool:</strong> The <strong>Zoom to full extents</strong> tool is a quick way to re-center the map by zooming out to the full geo-graphical extent of
the map layers in the map window. Web layers do not affect the map extents.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-8aec7046.png" /><strong> Add Data Tool:
</strong>Click the <strong>Add Data</strong> tool<strong>,</strong> the <strong>Browse for files</strong> browser<strong> </strong>will open. This
browser allows users to navigate to layers (including <em>&#42;.shp</em>, <em>&#42;.flt</em>, <em>&#42;.tif</em>, and <em>&#42;.vrt</em>) of interest and add the
layer(s) to the map window.
If the user clicks the down arrow next to the <strong>Add Data</strong> tool a shortcut menu will appear which allows the user to add data, add web
imagery, or create new data that is added to the map window.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-878d4d2c.png" /><strong> Query Features Tool (Q):
</strong>The <strong>Query Features </strong>tool identifies the geographic feature or place currently selected in the map window. Alternatively,
multiple features can be identified by dragging a box over the desired area. Once a selection is made the <strong>Feature Query</strong> dialog box
will open with the information presented in table format separated by <strong>Feature</strong> type. Only visible layers in the map window will be
queried and the <strong>Selectable Layers</strong> option does not modify the <strong>Query Features</strong> tool.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-ca04fae1.png" /><strong> Measure Distance Tool (M):
</strong>The <strong>Measure Distance</strong> tool allows the user to measure the distance along a specified line. With the tool selected, click to
initiate the measure line. Each subsequent click will add a new point to the measure line. When finished, double-click on the location of the final
point. The <strong>Measure Results</strong> dialog box will open and display the distance in the units the user has chosen (feet, miles, meters, or
kilometers).

Layers, refer to [Add Features to the Destination Locations Map
Layer](./18-appendix-map-preferences.mdx#add-features-to-the-destination-locations-map-layer).

#### Map Window Editor Toolbar

The set of tools located in the <strong>Editor Toolbar</strong> become active after users initiate an editing session for a map layer. When a user
right-clicks on a map layer in the <strong>Map</strong> <strong>Layers</strong> tab, from the shortcut menu (<FigReference figKey="figure-229" />),
click <strong>Edit</strong>, and the tools in the <strong>Editor Toolbar</strong> become active. Only one map layer can be edited at one time. For
lines and polygons, there are two editing modes: feature edit, and vertex edit. Clicking on a selected feature puts the user into the feature edit
mode (move, delete, reverse), double-clicking on a selected feature puts the user in the vertex edit mode (add, delete, edit vertex points).

  figKey="figure-234"

There are some tools available during an edit session which are not available from the shortcut menu (<FigReference figKey="figure-229" />) or the

<strong>Editor Toolbar</strong>. Clicking <strong>Delete</strong> will delete all currently selected features or currently selected vertices depending
 on which mode the user is in. Clicking the <strong>R</strong> key will reverse the direction of a currently selected line feature. Also,
right-clicking during an edit session allows the user to do multiple tasks depending on what is clicked.

</strong>The <strong>Add new features</strong> tool allows the user to add a new feature to the editable map layer by left-clicking to add feature
data. Selected features can be deleted by either clicking the <strong>Delete</strong> key, or right-click on the selected feature(s) and from the

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-7398525a.png" /><strong> Insert vertex Tool (I):
</strong>The <strong>Insert vertex</strong> tool allows the user to add a vertex to an existing line or polygon map layer. The <strong>Insert vertex
Tool</strong> becomes available when the selected feature is in the vertex edit mode. As the cursor gets close to the line or polygon edge the map
window will display what the updated feature will look like if a vertex is added at the cursor. Left-click to insert the new vertex.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-e9d16d85.png" /><strong> Break line feature into two
line features at a defined location Tool (K): </strong>The <strong>Break line feature</strong> tool will split a line feature into two features at the
 point the user clicked on the line. The user needs to be in the feature edit mode for this tool to be enabled.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-cf9d34fc.png" /><strong> Continue from end (lines
only) Tool: </strong>The <strong>Continue from end (lines only)</strong> tool continues a line feature from the end point of the line feature to a
second point clicked in the map window and each new click adds a vertex point to the line feature. Double-click to finish the line or right-click and
choose <strong>Finish Editing Vertices</strong>. The user needs to be in the vertex edit mode to enable this tool.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-3da2817d.png" /><strong> Add new line or polygon to
existing feature Tool: </strong>The <strong>Add new line or polygon to existing feature</strong> tool will allow the user to add a polygon or line to
an existing selected polygon or line feature. For example, to add a hole to a polygon, click to add points defining the hole and double-click to
finish. The user needs to be in the vertex edit mode for this tool to be enabled.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-d15e7265.png" /><strong> Feature Snapping Settings
Tool: </strong>The <strong>Feature Snapping Settings</strong> tool allows the user to define snapping rules when editing. Snapping will automatically
move the desired action to the nearest snap point when the cursor is close. This can be very useful when trying to line up endpoints of line features
or place points directly on top of other features. Enable snapping by clicking <strong>Enable Snapping</strong>. From the <strong>Snap to
Feature</strong> list, select the desired feature to snap to, then select how snapping will be applied (all points, end points, or edges).

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-25fae2d8.png" /><strong> Undo Tool (Ctrl + Z):
</strong>The <strong>Undo</strong> tool allows the user to undo changes made during an edit session. Only enabled if there are edits that can be
undone.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-d39f7782.png" /><strong> Redo Tool (Ctrl + Y):
</strong>The <strong>Redo</strong> tool allows the user to redo changes made. Only enabled if there are edits to redo.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-8d7596cf.png" /><strong> Save Edits Tool (Ctrl + S):
</strong>The <strong>Save Edits</strong> tool will save all edits made during the editing session. Only enabled if edits have been made.

- <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-d8eb8f4c.png" /><strong> Stop Editing Tool:
</strong>The<strong> Stop Editing</strong> tool will stop the edit session. At this point a new map layer can be selected for editing. If edits have
been made, then the user will be prompted to save them before stopping the edit session.

## Map Layers

Geographic Information System (GIS) files are the map layer formats supported by LifeSim and include: <em>\*.shp</em>, <em>\*.flt</em>, <em>\*.tif</em>,
and <em>\*.vrt</em>. The user can add map layers, add background map layers, remove map layers, adjust a map layer's position in the view, edit a map
layer's properties, zoom into a layer, and many other functions. The ability to manipulate the different map layers is specific for that map layer
format.

### Adding Map Layers

To add a map layer:

1. From the LifeSim main window (<FigReference figKey="figure-228" />), from the Mapping menu (<FigReference figKey="figure-237" />), click Add Map
   Layer. Also, from the General Toolbar ([Map Window Toolbars](./18-appendix-map-preferences.mdx#map-window-toolbars)), click . Either way, the Browse
   for files browser (<FigReference figKey="figure-238" />) will open.

1. From the <strong>Browse for files</strong> browser (<FigReference figKey="figure-238" />), navigate to the GIS file of interest that will be added
   as a map layer. Select the appropriate GIS file (<em>\*.shp</em>, <em>\*.flt</em>, <em>\*.tif</em>, or <em>\*.vrt</em>) and the <strong>File
   name</strong> box (<FigReference figKey="figure-238" />) will now display the selected filename.

1. Click <strong>Open</strong>, the <strong>Browse for files</strong> browser (<FigReference figKey="figure-238" />) will close, the selected dataset
   will now be displayed in the map window, and selected map layer name will appear in the <strong>Map Layer Tree</strong>.

### Adding Background Internet Maps

Knowing where the study is located in reference to known geographic positions can be quite helpful with communication. There are two types of internet
background maps, <strong>OpenStreetMap</strong> and <strong>Esri World Imagery</strong> map and there are two ways to add these background internet
maps.

To add a background internet map to the LifeSim study, from the Mapping menu (<FigReference figKey="figure-239" />) on the LifeSim main window (
{"\n"}<FigReference figKey="figure-233" />), point to Add Web Layer, and select the internet map from the available list which would be a good
background map for the study area. Also, from the General Toolbar ([Map Window Toolbars](./18-appendix-map-preferences.mdx#map-window-toolbars)),
click the Add Data dropdown menu (<FigReference figKey="figure-239" />), point to Add Web Imagery (<FigReference figKey="figure-239" />), and
select the internet map from the available list which would be a good background map for the study area. Either way, the selected background map
displays in the map window (<FigReference figKey="figure-240" />).

### Map Layer Position in the Map Layer Tree

Once a map layer is added to a LifeSim study, the user can adjust the position of a map layer. Within the LifeSim framework, some map layers represent
a particular LifeSim dataset (i.e., hydraulic data, structure inventories, emergency planning zones), while other map layers provide information.

The <strong>Map Layer Tree</strong> (<FigReference figKey="figure-240" />) displays the map layers from top to bottom, where the top-most layer
overlays the next one down in the list and continues to the bottom layer. In other words, if a background internet map is added, from the <strong>Add
Data</strong> dropdown menu <FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-0211c61a.png" /> (
{"\n"}<FigReference figKey="figure-239" />), that map layer will automatically be added to the bottom of the <strong>Map Layers Tree</strong> (
{"\n"}<FigReference figKey="figure-240" />), and will be underneath all layers listed above the internet map. At times it may be desirable to have various
map layers drawn in a different order in the map window. To move map layers:

1. From the <strong>Map Layers Tree</strong> (<FigReference figKey="figure-237" />), right-click on the map layer of interest, from the shortcut menu
   (<FigReference figKey="figure-229" />), either select <strong>Move Up</strong> or <strong>Move Down</strong>.

1. To move the map layer of interest below other map layers listed in the <strong>Map Layers Tree</strong>, click <strong>Move Down</strong>. The
   selected map layer will move one step below its current position. Repeat until the map layer of interest is at the desired location within the
   <strong>Map Layers Tree</strong>.

   {"\n"}<FigReference figKey="figure-272" />), from the <strong>Simplification Method</strong> list (<FigReference figKey="figure-272" />), select the
   appropriate simplification method. Each method has its own set of inputs. The simplification methods are briefly described:

- <strong>Visalingam Whyatt</strong> – An area based algorithm to eliminate points based on each individual points effective area (area within the
polygon the point effects). The point(s) which affect the polygon the least will be removed first. The user must enter the maximum number of points
(greater than four and less than two points than the total number of points in the polygon) to be eliminated (<strong>Number of Points</strong> box,
{"\n"}<FigReference figKey="figure-272" />). If the<strong> </strong>number of points entered is greater than the total number of points in the
polygon then an identical copy of the polygon will be made.

- <strong>Lang</strong> – A perpendicular distance based algorithm to eliminate points based on the perpendicular distance of the points. The
perpendicular distance is compared to a user entered tolerance limit for intermediate points between the end point and the entered number of look
ahead points. The algorithm works based on the simplify inputs (<strong>Number of Points</strong> box, <FigReference figKey="figure-272" />),
tolerance and look ahead points that the user enters. The <strong>Tolerance</strong> value must be greater than zero and the <strong>Look Ahead
Points</strong> number must be less than or equal to the total number of points in the polyline or polygon shapefile. The point(s) which have a
perpendicular distance less than the <strong>Tolerance</strong> limit for a set number of <strong>Look Ahead Points</strong> will be eliminated until
there are no more intermediate points. If the<strong> </strong>number of <strong>Look Ahead Points</strong> entered is greater than the total number
of points in the polygon or polyline shapefile then an identical copy of the selected shapefile will be made.

- <strong>Douglas Peucker</strong> – Similar to the <strong>Lang</strong> method, in that the points are eliminated based on the points distance from
 a simplification line. The simplification line is a straight line between key points. Key points are set for points greater than the
<strong>Tolerance</strong> level defined by the user. The <strong>Tolerance</strong> level is the <strong>Simplify Inputs</strong> for the
<strong>Douglas Peucker</strong> method, and the value must be greater than zero. In the end only the key points will remain after this simplification
 method is used.

1. An optional step, Simplify Only Selected Features (<FigReference figKey="figure-272" />) allows the user to save the new shapefile with only
   features that have been selected for the selected map layer (i.e., using the Select By Attribute option explained in [Map Layer Attributes Dialog
   Box](./18-appendix-map-preferences.mdx#map-layer-attributes-dialog-box), or the Select options explained in [Map Layer Select
   Options](./18-appendix-map-preferences.mdx#map-layer-select-options)).

1. Click <strong>OK</strong>, a <strong>Verify Add Results</strong> message window will open (<FigReference figKey="figure-265" />) asking the user

study to represent possible evacuation locations during flooding events. The destination data is represented as points in space, LifeSim will find the
nearest road from a destination point and consider that the evacuation location. In other words, destination points do not need to connect with a
road for LifeSim simulations to be successful.
