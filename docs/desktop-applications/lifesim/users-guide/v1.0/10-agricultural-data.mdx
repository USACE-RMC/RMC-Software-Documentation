# Agricultural Data

New to LifeSim Version 2.0 is <strong>agricultural data</strong>. Agriculture data was added for the purpose of estimating economic impacts from
damage to existing crops and interruptions to the planting, growing, and harvesting of crops due to flooding in agricultural areas. Agricultural data
is not typically used in a consequence assessment; however, it may assist the user in understanding the amount and type of agriculture in the
floodplain and potential flood impacts to it. To compute agricultural damages hydraulic data (including duration and event arrival), agricultural
data, crop budgets, crop values, and crop duration-damage relationships are required. Review the LifeSim Version 2.0 Technical Reference Manual,
Chapter 7, for more information regarding the agricultural inventory and the computational procedures used to calculate agricultural flood damages.

## Import Agricultural Data

Users can import agricultural data directly from the U. S. Department of Agriculture (USDA) National Agricultural Statistics Service (NASS) Cropland
Data Layer (CDL) database through LifeSim (Version 2.0). Alternatively, users can download a NASS GeoTiff file from the NASS website: . The NASS
website provides the CDL in GeoTiff (<em>&#42;.tif</em>) format, where each cell represents a crop type (review the HEC-FIA Version 3.0 User’s Manual,
Appendix B for tips preparing NASS data for import into LifeSim). Then users can import the prepared NASS GeoTiff file into LifeSim (Version 2.0).
This chapter describe the ways agricultural data in different formats can be added to, edited, and displayed in an LifeSim study.

## Edit Agricultural Grid

When completing an agricultural economic loss assessment in LifeSim, Economic Guidance Memorandum (EGM) 14-02 states that current normalized
agricultural prices can be found at the U.S. Department of Agriculture (USDA) website. The EGM also states that the USACE website will no longer
disseminate current normalized agricultural prices, but its location will be provided as a memorandum. Each crop category selected (from the
<strong>Crop Editor</strong>, <FigReference figKey="figure-153" />) to be used in the damage compute requires planting date information and monthly
crop production costs. Typically, crop planting and production information can be obtained from a local land-grant university through an agricultural
extension agency. For example, the USDA National Institute of Food and Agriculture’s land-grant University directory. Additionally, price, yield, and
harvest cost and date information need to be populated. This information typically comes from the agriculture extension service or from the NASS
website.

Note, when agriculture data is imported with the option to <strong>Use Default Crop Functions</strong>  (<FigReference figKey="figure-148" /> and
{"\n"}<FigReference figKey="figure-149" />) then several crops in the <strong>Crop Editor</strong> (e.g., <em><strong>Alfalfa</strong></em> in
{"\n"}<FigReference figKey="figure-153" />) contain pre-populated crop budgets, crop values, and crop duration-damage relationships.

### Include Crop in Damage Compute

1. Navigate through the list of crops in the <strong>Crop Data Tree</strong> (<FigReference figKey="figure-153" />), and check  the <strong>Use in
Damage Compute</strong> checkbox to select the crops to include in the agricultural flood damages calculated during the economic damage assessment. By
 default, all crops listed in the <strong>Crop Editor</strong> (<FigReference figKey="figure-153" />) have the <strong>Use in Damage Compute</strong>
 checkbox  unchecked; therefore, the user must select the specific crops to include in the damage assessment.

1. After selecting the crop type from the <strong>Crop Data Tree</strong> and enabling the <strong>Use in Damage Compute</strong> option, enter (or
edit default) crop functions (planting, harvesting, and duration percent-damage relationships) in the editing panel.

### Define the Crop Budget

1. Enter (or edit) the crop planting data. Click
{"\n"}<FigureInline src="figures/desktop-applications/lifesim/users-guide/v1.0/inline-images/inline-d3ff7467.png" /> to select the <strong>Start
Planting</strong> date and enter the <strong>Planting Window (days)</strong> in the textbox (<FigReference figKey="figure-153" />). The start
planting date and planting window define the first and last possible plantings for the crop. Define the monthly crop planting costs by entering either
 a <strong>Fixed Monthly Costs ($)</strong> value, or by completing the <strong>Variable Monthly Costs ($)</strong> table (
{"\n"}<FigReference figKey="figure-153" />).

1. If desired, check  the <strong>Substitute Crop</strong> checkbox to enable the dropdown menu, and select a secondary crop (
{"\n"}<FigReference figKey="figure-153" />). The <strong>Substitute Crop</strong> option allows users to identify a secondary crop which LifeSim will
use during the agricultural flood damages calculation if flooding is determined to limit the ability to plant the primary crop. Review the LifeSim
Version 2.0 Technical Reference Manual, Chapter 7, for more information regarding the computational procedures used to calculate agricultural flood
damages. <strong>NOTE:</strong> When using a substitute crop, users must include the secondary crop in the damage compute and define the crop
functions.

### Define Crop Harvest Data

1. From the <strong>Harvest Information</strong> panel (<FigReference figKey="figure-153" />) enter the days required to harvest the crop, the yield
(units per acre), the unit price, the harvest cost (per acre), and the percentage of the total crop value lost due to late planting. To enter the
harvest data, select the textbox and enter the appropriate value.

### Define Crop Percent-Damage Relationships

1. The <strong>Duration Percent-Damage Relationships</strong> panel (<FigReference figKey="figure-153" />) provides a table for defining the crop
damage curve. The damage curve defines the percentage of crop loss per month based on the duration (i.e., the number of days) crops are inundated each
 month.

1. By default, un-defined crops contain one row for adding a duration percent-damage relationship. However, users can add damage curves by clicking
the <strong>Add Duration Row</strong> (<FigReference figKey="figure-153" />) button. Double-click a cell in the table to enter the duration flooded
(days) and/or percent crop loss per month. Alternatively, right-click a row and from the shortcut menu (<FigReference figKey="figure-127" />), click
<strong>Insert Row(s)</strong> to insert a row above the selected row or click <strong>Add Row(s)</strong> to add a row at the bottom of the table.

1. To remove damage curves, select the appropriate row and click the <strong>Delete Bottom Duration Row</strong> (<FigReference figKey="figure-153" />
) button. Alternatively, right-click the appropriate row and from the shortcut menu (<FigReference figKey="figure-127" />), click <strong>Delete
Row(s)</strong> to remove the selected row.

1. Click <strong>OK</strong> to close the <strong>Crop</strong> <strong>Editor</strong> (<FigReference figKey="figure-153" />) and save all entered
crop data.
