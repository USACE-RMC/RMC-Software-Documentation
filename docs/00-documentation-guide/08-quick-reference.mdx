# Quick Reference

This page provides quick access to commonly used commands, syntax, and solutions for everyday documentation tasks.

---

## Common Tasks Cheat Sheet

### Starting the Development Server

```bash
npm start
```

Opens the site at `http://localhost:3000` with live reload.

### Building for Production

```bash
npm run build
```

Creates production-ready static files in the `build/` folder.

### Running Build Scripts

```bash
# Regenerate all build artifacts
npm run counters    # Update figure/table/equation numbers
npm run sidebars    # Regenerate sidebar navigation
npm run versions    # Update version metadata
```

### Adding a New Document

1. Navigate to the appropriate version folder (e.g., `docs/lifesim/v1.0/`)
2. Create a new `.mdx` file with prefix: `01-my-new-document.mdx`
3. Add front matter:
   ```mdx
   ---
   title: My New Document
   ---
   ```
4. Write your content using Markdown and React components
5. Save the file - sidebar updates automatically

### Adding a New Software Manual

1. Create folder structure:

   ```
   docs/
   └── my-software/
       ├── v1.0/
       │   ├── 01-introduction.mdx
       │   └── 02-getting-started.mdx
       └── category.json
   ```

2. Create `category.json`:

   ```json
   {
     "label": "My Software",
     "position": 3
   }
   ```

3. Run sidebar script:
   ```bash
   npm run sidebars
   ```

### Inserting a Figure

```mdx
import Figure from '@site/src/components/Figure';

<Figure figKey="my-figure" caption="This is my figure caption" src="/figures/my-software/v1.0/my-image.png" alt="Description of image" />
```

**Reference the figure:**

```mdx
import FigureReference from '@site/src/components/FigureReference';

As shown in <FigureReference figKey="my-figure" />, the process...
```

### Creating a Table

```mdx
import TableVertical from '@site/src/components/TableVertical';

<TableVertical
  tableKey="my-table"
  caption="Table caption here"
  header={['Column 1', 'Column 2', 'Column 3']}
  data={[
    ['Row 1 Col 1', 'Row 1 Col 2', 'Row 1 Col 3'],
    ['Row 2 Col 1', 'Row 2 Col 2', 'Row 2 Col 3'],
  ]}
/>
```

### Adding an Equation

```mdx
import Equation from '@site/src/components/Equation';

<Equation equationKey="my-equation" caption="Caption for equation" equation="E = mc^2" />
```

**Reference the equation:**

```mdx
import EquationReference from '@site/src/components/EquationReference';

As shown in <EquationReference equationKey="my-equation" />, energy equals...
```

### Adding Citations

```mdx
import Citation from '@site/src/components/Citation';
import CitationFootnote from '@site/src/components/CitationFootnote';

This is cited text <Citation citationKey="smith2020" />.

At the end of the page:

<CitationFootnote />
```

### Cross-Referencing

```mdx

See [Introduction](../01-introduction.mdx#overview) for the overview section.
```

## File Naming Conventions

### Document Files

| Pattern          | Example                  | Purpose                 |
| ---------------- | ------------------------ | ----------------------- |
| `00-` to `99-`   | `01-introduction.mdx`    | Chapter ordering prefix |
| Kebab-case       | `03-getting-started.mdx` | Lowercase with hyphens  |
| `.mdx` extension | `04-configuration.mdx`   | MDX format required     |

### Image Files

| Pattern                | Example                        | Purpose              |
| ---------------------- | ------------------------------ | -------------------- |
| Descriptive names      | `workflow-diagram.png`         | Clear purpose        |
| Version-specific paths | `/figures/lifesim/v1.0/`       | Organized by version |
| Supported formats      | `.png`, `.jpg`, `.svg`, `.gif` | Standard web formats |

### Component Keys

| Pattern                | Example                          | Purpose                    |
| ---------------------- | -------------------------------- | -------------------------- |
| Unique within document | `figKey="installation-screen"`   | No duplicates in same file |
| Descriptive            | `tableKey="performance-metrics"` | Clear identification       |
| Kebab-case             | `equationKey="flood-frequency"`  | Consistent formatting      |

### Folder Structure

```
docs/
└── software-name/
    ├── category.json        # Required for navigation
    ├── v1.0/                # Version folder
    │   ├── 01-intro.mdx
    │   └── 02-chapter.mdx
    └── v1.1/                # New version folder
        ├── 01-intro.mdx
        └── 02-chapter.mdx
```

**Citations:**

**Content Organization:**

**Multimedia:**

**Navigation:**

- `Button` - Styled buttons and links

## Troubleshooting Quick Fixes

### Build Won't Start

**Error:** `npm start` fails

**Solutions:**

```bash
# Clear cache and rebuild
rm -rf .docusaurus
rm -rf node_modules
npm install
npm start

# Windows PowerShell:
Remove-Item -Recurse -Force .docusaurus
Remove-Item -Recurse -Force node_modules
npm install
npm start
```

### Images Not Showing

**Problem:** Image not displaying on page

**Solutions:**

1. Check path starts with `/figures/` not `figures/`
2. Verify file exists in `static/figures/` folder
3. Check file extension matches (`.png` vs `.PNG`)
4. Clear browser cache (Ctrl+Shift+R / Cmd+Shift+R)

### Numbering Wrong

**Problem:** Figure/table/equation numbers are incorrect

**Solution:**

```bash
npm run counters
```

This regenerates all numbering based on current document structure.

### Sidebar Not Updating

**Problem:** New document doesn't appear in sidebar

**Solution:**

```bash
npm run sidebars
```

This regenerates the sidebar configuration from your document structure.

### Component Import Error

**Error:** `Module not found: Can't resolve '@site/src/components/Figure'`

**Solutions:**

1. Check import path is correct: `@site/src/components/ComponentName`
2. Verify component name matches exactly (case-sensitive)
3. Restart dev server: Stop (Ctrl+C) and run `npm start` again

### Changes Not Appearing

**Problem:** Saved changes don't show in browser

**Solutions:**

1. Check terminal for build errors
2. Hard refresh browser (Ctrl+Shift+R / Cmd+Shift+R)
3. Restart dev server
4. Clear `.docusaurus` cache:
   ```bash
   rm -rf .docusaurus
   npm start
   ```

### Cross-Reference Broken

**Problem:** FigureReference/TableReference shows error

**Solutions:**

1. Verify `figKey` / `tableKey` matches exactly
2. Check both components are in same document
3. Run counters script: `npm run counters`
4. Restart dev server

### Git Push Rejected

**Error:** `! [rejected] main -> main (fetch first)`

**Solution:**

```bash
# Pull latest changes first
git pull origin main

# Resolve any conflicts, then push
git push origin main
```

## Common Patterns

### Adding a New Chapter

1. Determine next chapter number (e.g., `05-`)
2. Create file: `docs/software/v1.0/05-my-chapter.mdx`
3. Add front matter:
   ```mdx
   ---
   title: My Chapter Title
   ---
   ```
4. Save - sidebar updates automatically

### Moving/Renaming a Document

1. Rename file in VS Code or file explorer
2. Update any internal links pointing to old filename
3. Run sidebar script: `npm run sidebars`

### Creating a New Version

1. Copy version folder:

   ```bash
   cp -r docs/software/v1.0 docs/software/v1.1
   ```

2. Update documents in new version folder

3. Run version script:
   ```bash
   npm run versions
   ```

### Converting from Word

1. Ensure DOCX uses RMC template
2. Prepare bibliography JSON file
3. Run converter in development mode:
   ```bash
   python scripts/convert_DOCX_MDX/main.py
   ```
4. Review output in `docs/` folder
5. Make manual adjustments as needed

## Quick Decision Trees

### Which component should I use?

**For images:**

- Need caption and number? → `Figure`
- Just display image? → `FigureInline` or `FigureNoRef`

**For tables:**

- Standard data table? → `TableVertical`
- Row-based comparison? → `TableHorizontal`
- Acronym list? → `TableAcronyms`

**For equations:**

- Need to reference later? → `Equation`
- One-time display? → `EquationNoRef`
- Inline variable? → `EquationChip` or inline LaTeX `$x$`

**For organizing content:**

- Step-by-step process? → `ProcessList`
- Multiple alternatives? → `DocTabs`
- Optional details? → `CollectionList`

**For navigation:**

- Dashboard tiles? → `ContentBubble` or `CategoryList`
- Button/link? → `Button` or `NavLink`

### Where should I put my file?

**New document:**
→ `docs/software-name/version/##-filename.mdx`

**New image:**
→ `static/figures/software-name/version/filename.png`

**New bibliography:**
→ `static/bibliographies/software-name/version/bibliography.json`

**Source document:**
→ `static/source-documents/software-name/version/original.docx`
