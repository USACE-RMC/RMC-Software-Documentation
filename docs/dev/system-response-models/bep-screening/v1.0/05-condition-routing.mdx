# Condition Routing

The screening model determines one of six site conditions based on three Boolean observations. Each condition defines which sub-model calculations are performed and how the event tree nodes are populated for the toe and daylighting exit paths.

## Condition Determination

Three Boolean flags fully determine the site condition. The `DetermineCondition` method in `BEPScreeningModel.cs` evaluates these flags in sequence.

Conditions 1 and 2 produce identical SRP values because both set P(unfiltered exit) and P(initiation) to 1.0. The distinction between them exists for condition tracking and reporting purposes, but the event tree mathematics are equivalent.

### Trigger Elevation Override

For heave conditions (4 and 6), the `ApplyHeaveWithTrigger` method applies a headwater-dependent override to the initiation probability:

<EquationNoRef equation="P_{\text{init}}(h) = \begin{cases} 1.0 & \text{if } h \geq h_{\text{trigger}} \\ P_{\text{BTCase4}}(h) & \text{if } h < h_{\text{trigger}} \end{cases}" />

where $h_{\text{trigger}}$ is the trigger elevation (`TriggerElevationFt`). This reflects the engineering judgment that boils have been historically observed at or above this pool elevation, making initiation certain at those levels.

## Daylighting Exit Path Node Assignment

The daylighting exit path is only evaluated for conditions 5 and 6, where the confining blanket terminates at a known distance downstream.

For the daylighting exit path, both P(unfiltered exit) and P(initiation) are set to 1.0 because the blanket has terminated &mdash; there is no confining layer at the daylighting exit point to prevent seepage from reaching the surface or to resist piping initiation.

## Geometry Computations

The condition determines how geometric parameters are computed for the sub-models.

### Seepage Path Lengths

The model computes seepage path lengths for the Schmertmann progression model based on the exit path:

<EquationNoRef equation="L_{\text{toe}} = L_2 = \text{EmbankmentBaseWidth} + \text{BermLength}" />

<EquationNoRef equation="L_{\text{daylight}} = L_2 + \text{DaylightingExitDistanceFt}" />

A longer seepage path results in a lower applied gradient ($i_{\text{adv}} = \Delta H / L$), which generally produces a lower probability of progression. The daylight path has a longer seepage path than the toe path, so the daylight progression probability is typically less than or equal to the toe progression probability.

For conditions 1 and 2 (no confining layer), a "dummy" BT Case 4 run is executed solely to obtain the expanded headwater and tailwater arrays. The initiation and gradient results from this run are not used.

For blowout conditions (3 and 5), the confining layer's saturated unit weight is used because the critical gradient represents the blanket rupture threshold. For heave conditions (4 and 6), the foundation sand's saturated unit weight is used because the critical gradient represents the sand's resistance to heave at the boil location.
