## Modules

### [Screening Management](./screening-management)

Core screening entities, team assignments, project information, and URL routing. **4 tables:**

- **Screenings** — Top-level entity representing a complete dam screening assessment (14 columns)
- **Users** — User accounts with authentication and role management (14 columns)
- **ScreeningAssignments** — Maps users to screenings with role-based access (6 columns)
- **ProjectInformation** — Dam identification, location, geometry, and NID data (50 columns)

### [Authentication & Notifications](./auth-system)

Account lifecycle and user notification management. **3 tables:**

- **AccountRequests** — Account registration requests with admin approval workflow (11 columns)
- **PasswordResetRequests** — Password reset requests with admin approval (6 columns)
- **Notifications** — User notification records for system events (6 columns)

### [Site Management](./site-management)

System-wide alerts and informational ticker messages. **2 tables:**

- **SiteAlerts** — Page-targeted alert banners with severity levels (9 columns)
- **TickerMessages** — Scrolling informational messages on the dashboard (8 columns)

### [PFM-01: Overtopping Erosion](./pfm-01)

Overtopping erosion failure mode evaluation with the three-table SRP pattern. **4 tables:**

- **Pfm01** — PFM-01 location entity (35 columns)
- **Pfm01_DstSrp** — DST-calculated SRP curves and nodal probabilities
- **Pfm01_UserSrp** — User-defined SRP curves
- **Pfm01_ActiveSrp** — Active SRP cache synced from DstSrp or UserSrp
