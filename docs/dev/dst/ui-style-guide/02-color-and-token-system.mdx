# Color and Token System

The DST uses a structured token system built on Tailwind CSS custom properties. All colors are defined as global tokens in `frontend/src/app/index.css` and consumed via Tailwind classes.

## Token Architecture Rules

<ProcessList
  items={[
    { title: <><strong>Global tokens in <code>index.css</code></strong> define the palette — they describe color values, not component behavior.</> },
    { title: <><strong>Component-level constants</strong> at the top of each file map global tokens to component-specific roles using Tailwind class names.</> },
    { title: <><strong>No arbitrary hex values in components.</strong> If you're typing <code>#</code> inside a component, it should become a Tailwind class or a new global token. Seeing <code>text-[#94a3b8]</code> in a component means there's a gap in the token system that should be filled.</> },
    { title: <><strong>Nav-specific tokens do not belong in <code>index.css</code>.</strong> Tab hover backgrounds, icon colors, and active states are component concerns owned by the component's constants block.</> },
  ]}
  numberStyle="bubble"
/>
