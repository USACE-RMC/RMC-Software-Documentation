## Auto-Save Pattern

The application uses debounced auto-save for user inputs. No "Save" button for normal parameter entry.

## Chart Styling

Charts use Chart.js with consistent styling. All chart colors are resolved from CSS custom properties at render time.

```javascript
import { getChartColors, FONT_FAMILY } from '@components/common/chart/utils';

const C = getChartColors();

// Axis styling
ticks: { color: C.text, font: { family: FONT_FAMILY, size: 15 } }
grid:  { color: C.gridMinor, drawBorder: false }
title: { color: C.text, font: { family: FONT_FAMILY, size: 16 } }

// Tooltip styling
tooltip: {
  backgroundColor: C.tooltipBg,
  borderColor: C.tooltipBorder,
  borderWidth: 1,
  titleColor: C.text,
  bodyColor: C.text,
}
```

**Font family:** `'Aptos Narrow, system-ui, sans-serif'` (from the `FONT_FAMILY` constant)

**Line colors:** Use `C.primary` (steel blue) for DST estimates, `C.revised` (amber) for user-modified values.

## Do's and Don'ts

<DocTabs
  items={[
    {
      title: "Do",
      label: "Do",
      child: (
        <ul>
          <li>Use <code>Section</code> for new page sections (not <code>Panel</code>)</li>
          <li>Use Tailwind's <code>slate</code> scale for neutrals</li>
          <li>Declare visual constants at the top of component files</li>
          <li>Use <code>dst-*</code> tokens for brand colors</li>
          <li>Test at all 5 reference device sizes</li>
          <li>Keep page components thin — extract complex JSX to sub-components</li>
        </ul>
      ),
    },
    {
      title: "Don't",
      label: "Don't",
      child: (
        <ul>
          <li>Use arbitrary hex values in components (<code>text-[#94a3b8]</code>)</li>
          <li>Use red for anything other than errors/warnings/USACE branding</li>
          <li>Put business logic in page components</li>
          <li>Create custom tokens for neutrals (use <code>slate-*</code>)</li>
          <li>Skip the visual constants pattern</li>
          <li>Use inline styles — use Tailwind classes</li>
        </ul>
      ),
    },
  ]}
/>
