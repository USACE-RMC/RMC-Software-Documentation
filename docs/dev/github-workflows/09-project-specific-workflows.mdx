# Project-Specific Workflows

## Web Applications (React + ASP.NET)

**Special considerations:**

- Test both frontend and backend changes together
- Check browser console for errors
- Verify API contracts haven't broken
- Test responsiveness on different screen sizes
- Check database migrations if schema changed

**PR checklist additions:**

<ProcessList
  items={[
    { title: 'Frontend builds without errors (`npm run build`)' },
    { title: 'Backend builds without errors' },
    { title: 'API endpoints tested (Postman, Swagger, or manual)' },
    { title: 'Database migrations tested' },
    { title: 'No console errors or warnings' },
  ]}
  numberStyle="plain"
/>

## Desktop Tools (WPF)

**Special considerations:**

- Test on multiple Windows versions if possible
- Check for memory leaks in long-running operations
- Verify file I/O operations don't block UI
- Test with large datasets if applicable

**PR checklist additions:**

<ProcessList
  items={[
    { title: 'Application starts and closes cleanly' },
    { title: 'No UI freezing during operations' },
    { title: 'Settings persist correctly' },
    { title: 'File operations handle errors gracefully' },
  ]}
  numberStyle="plain"
/>

## Libraries/Packages (C# Models)

**Special considerations:**

- These are used by other projects - breaking changes are critical
- Consider semantic versioning implications
- Document public APIs clearly
- Think about backward compatibility

**PR checklist additions:**

<ProcessList
  items={[
    { title: 'XML documentation added for public members' },
    { title: 'Breaking changes clearly marked' },
    { title: 'Examples provided for new functionality' },
    { title: 'Considered impact on dependent projects' },
  ]}
  numberStyle="plain"
/>

**Versioning:**

- **Patch** (0.0.X): Bug fixes, no API changes
- **Minor** (0.X.0): New features, backward compatible
- **Major** (X.0.0): Breaking changes

## Data Science Tools (Python, R)

**Special considerations:**

- Document data sources and assumptions
- Include example data or test cases
- Note Python/R version requirements
- Consider reproducibility

**PR checklist additions:**

  items={[

  numberStyle="plain"
