# Dam Information

The Dam Information page collects the physical characteristics of the embankment dam that the DST uses to evaluate potential failure modes during the screening-level risk assessment. The user describes the embankment's structural components, construction materials, cross-section geometry, and abutment conditions. These inputs feed directly into the failure mode screening and evaluation modules, where they influence the system response probability (SRP) calculations and inform the overall risk estimate.

Completing the Dam Information page is a prerequisite for running most failure mode evaluations. The data entered here establishes the physical baseline against which each potential failure mode is assessed.

## Overview

The Dam Information page is organized into five sections, presented vertically on the page (<FigReference figKey="dam-info-overview" />):

1. **Dam Overview** — Basic dam characteristics, populated from the National Inventory of Dams (NID) when available or entered by the user.
2. **Embankment Components** — Checkbox selection of structural features present in the embankment.
3. **Embankment Materials** — Material cards defining the soil and protection types used in the embankment.
4. **Embankment Cross Sections** — Geometry, slope data, and an interactive cross-section diagram for each representative cross section.
5. **Abutment Slopes** — Foundation profile and abutment slope angles.

## Dam Overview

The Dam Overview section displays basic dam characteristics (<FigReference figKey="dam-overview" />). When data from the National Inventory of Dams (NID) is available, the DST auto-populates these fields and marks each one with a blue **NID** badge. NID-sourced fields are read-only; fields without NID data are editable by the user.

<Figure
  figKey="dam-overview"
  src="figures/web-applications/dst/users-guide/v1.0/figures/figure-04-02.png"
  alt="The Dam Overview section showing two fields: Year Completed displaying 1960 and Dam Height (ft) displaying 50.0, each with a blue NID badge next to the label and a gray disabled background indicating the values were sourced from the National Inventory of Dams."
  caption="Dam Overview section with NID-sourced fields."
/>

The following fields are displayed:

- **Year Completed** — The year the dam was originally constructed. This value is used in failure mode evaluations that account for the age of the structure. When sourced from NID, a blue **NID** badge appears next to the label and the field is read-only.
- **Dam Height (ft)** — The structural height of the dam in feet. This is a critical parameter used extensively across failure mode evaluations, particularly for overtopping and internal erosion assessments. When sourced from NID, a blue **NID** badge appears next to the label and the field is read-only.

## Embankment Components

The Embankment Components section allows the user to identify the structural features present in the embankment (<FigReference figKey="embankment-components" />). The user checks all applicable components. These selections influence which failure modes are relevant and affect parameters in the failure mode screening and evaluation modules.

<Figure
  figKey="embankment-components"
  src="figures/web-applications/dst/users-guide/v1.0/figures/figure-04-02b.png"
  alt="The Embankment Components section showing six checkbox options arranged vertically: Conduit through Embankment (with a help icon), Vertical or Inclined Drain/Filter (checked, with a Top Elevation input field visible), Relief Well and Horizontal Drain, Core Trench, Cutoff or Seepage Barrier Wall (with a help icon), and Upstream Facing Element (with a help icon)."
  caption="Embankment Components section."
/>

The available components are:

- **Conduit through Embankment** — A pipe, culvert, or other conduit that passes through the body of the embankment dam, typically for water conveyance or outlet works. Selecting this component enables conduit-related failure modes during screening.
- **Vertical or Inclined Drain/Filter** — An internal drainage or filter element within the embankment. When checked, a **Top Elevation (ft)** input field appears, allowing the user to record the top elevation of the drain or filter. This component also adds a corresponding material card in the Embankment Materials section.
- **Relief Well and Horizontal Drain** — A horizontal drainage system at the base of the embankment. When checked, a corresponding material card appears in the Embankment Materials section.
- **Core Trench** — A trench excavated into the foundation and backfilled with impervious material to reduce seepage.
- **Cutoff or Seepage Barrier Wall** — A wall of concrete, sheet pile, slurry, or other impervious material extending into the foundation to reduce underseepage. When checked, a **Top Elevation (ft)** input field appears.
- **Upstream Facing Element** — A membrane or layer on the upstream face (e.g., concrete facing, geomembrane, or asphalt lining) that serves as the primary water barrier for the embankment.

## Embankment Materials

The Embankment Materials section defines the materials used in the embankment (<FigReference figKey="embankment-materials" />). Material definitions are used by the failure mode evaluation modules to assess internal erosion susceptibility, filter compatibility, and erosion resistance.

### User-Defined Materials

The user defines as many materials as needed to characterize the embankment. Each material card contains the following fields:

- **Description** — A free-text label to identify the material (e.g., "Impervious Core," "Random Fill"). This description appears in dropdowns elsewhere on the page where a material selection is required.
- **Material Type** — A dropdown selecting the USCS soil classification of the material. The available options are:
  - Silty Sand (SM)
  - Silty Gravel (GM)
  - Low Plasticity Silt (ML)
  - Clayey Sand (SC)
  - Clayey Gravel (GC)
  - Silt and Clay (CL-ML)
  - Lean Clay (CL)
  - High Plasticity Silt (MH)
  - Fat Clay (CH)
- **Gradation Data** — Indicates whether particle size distribution data from sieve analysis is available for this material. A help icon next to the label provides additional context. When set to "Yes," an **Enter Gradation** button appears, allowing the user to input the gradation curve in a modal dialog.

The user adds new materials by clicking the **Add Material** button. Materials can be removed by clicking the trash icon in the card header, provided at least one material remains.

### Component-Linked Materials

Certain material cards appear automatically based on the selections in the Embankment Components section. These cards have a lighter accent color to distinguish them from user-defined materials. The user cannot add or remove these cards; they appear and disappear as their corresponding component is checked or unchecked.

- **Vertical or Inclined Drain/Filter** — Appears when the Vertical or Inclined Drain/Filter component is checked. The Material Type dropdown offers filter-specific options: Sand (SP, SW), Gravelly Sand (SP, SW), Sandy Gravel (GP, GW), and Gravel (GP, GW).
- **Downstream Horizontal Drain** — Appears when the Relief Well and Horizontal Drain component is checked. Uses the same filter material type options.
- **Downstream Slope Protection** — Always displayed. The Material Type dropdown offers protection-specific options: Good Grass Cover, Average Grass Cover, Poor Grass Cover, Riprap, and None. When "Riprap" is selected and Gradation Data is set to "Yes," an **Enter Riprap D<sub>50</sub>** button appears instead of the standard gradation entry. The Description field is not editable for this card.

## Embankment Cross Sections

The Embankment Cross Sections section defines the geometry and slope data for each representative cross section of the embankment (<FigReference figKey="cross-sections-empty" />). The user may define one or more cross sections to represent different reaches of the dam. Each cross section captures crest geometry, core geometry (for zoned embankments), material assignments, and upstream/downstream slope profiles.

### Cross Section Properties

Each cross section contains the following fields:

- **Name or Station Range** — A text label identifying the cross section (e.g., "Valley Section," "Station 10+00 to 15+00"). This label appears in cross-section selection dropdowns on failure mode evaluation pages.
- **Foundation** — The type of foundation beneath this cross section. A help icon next to the label explains the distinction between foundation types. Options are "Soil Foundation" or "Rock Foundation." The foundation type affects downstream failure mode evaluations, as soil and rock foundations have different failure mechanisms for seepage and piping.
- **Embankment Type** — The structural configuration of the embankment. A help icon next to the label describes each option. Options are:
  - **Zoned** — An embankment with distinct material zones (e.g., impervious core with outer shells). Selecting "Zoned" enables the core geometry fields and core slope tables.
  - **Homogeneous Earthfill** — A single-material embankment. Core geometry and core slope fields are disabled.
  - **Other** — Non-standard embankment configurations.

### Crest Geometry

- **Crest Elevation (ft)** — The elevation of the dam crest. When entered, this value auto-populates the first row's Top Elevation in the upstream and downstream face slope tables.
- **Crest Width (ft)** — The width of the dam crest.

### Core Geometry (Zoned Embankments Only)

These fields are enabled only when Embankment Type is set to "Zoned":

- **Top of Core Elevation (ft)** — The elevation of the top of the impervious core. When entered, this value auto-populates the first row's Top Elevation in the upstream and downstream core slope tables.
- **Top Width of Core (ft)** — The width of the core at its top elevation.

### Material Properties

- **Core Material** (or **Primary Embankment Material** for homogeneous embankments) — A dropdown selecting one of the user-defined materials from the Embankment Materials section. This identifies the primary impervious material for the cross section.
- **Potential for Dispersive Soils** — Indicates whether the core (or embankment, for homogeneous types) contains dispersive soils. A help icon next to the label explains how to identify dispersive potential through pinhole tests, crumb tests, chemical analysis, or historical evidence. Options are "Yes" or "No."

### Slope Tables

Each cross section includes four slope tables that define the slope profile from crest to toe (<FigReference figKey="cross-sections-filled" />):

- **Upstream Embankment Face Slope** — The outer upstream slope of the embankment.
- **Downstream Embankment Face Slope** — The outer downstream slope of the embankment.
- **Upstream Core Slope** — The upstream slope of the impervious core (disabled for homogeneous embankments).
- **Downstream Core Slope** — The downstream slope of the impervious core (disabled for homogeneous embankments).

Each slope table has three columns:

| Column | Description |
|--------|-------------|
| **Top Elevation (ft)** | The elevation at the top of the slope segment. |
| **Bottom Elevation (ft)** | The elevation at the bottom of the slope segment. |
| **Slope (H:1V)** | The horizontal distance per one unit of vertical rise. For example, a value of 2.5 means 2.5 feet horizontal for every 1 foot of vertical height. A help icon in the column header provides this explanation. |

Multiple rows can be entered to define slope breaks (berms or changes in grade). The user adds rows using the toolbar buttons above each table: **Add row** appends a row at the bottom, **Insert row** inserts below the selected row, and **Delete row** removes the selected row.

### Cross-Section Diagram

When the user enters crest parameters and at least one upstream/downstream slope pair, an interactive SVG diagram appears to the right of the slope tables (<FigReference figKey="cross-sections-filled" />). The diagram updates in real time as slope data is modified, providing immediate visual feedback on the cross-section geometry. For zoned embankments, the core outline is shown as a dashed overlay within the embankment shell.

### Managing Multiple Cross Sections

The user adds additional cross sections by clicking the **Add Cross Section** button at the bottom of the section. Each cross section is numbered sequentially (Cross Section #1, #2, etc.) and separated by a horizontal rule. Cross sections can be deleted by clicking the trash icon next to the section heading, provided at least one cross section remains.

## Abutment Slopes

The Abutment Slopes section records the foundation and abutment profile for the dam (<FigReference figKey="abutment-slopes" />). These values are used in failure mode evaluations that assess slope stability and seepage along the abutment contacts.

<Figure
  figKey="abutment-slopes"
  src="figures/web-applications/dst/users-guide/v1.0/figures/figure-04-07.png"
  alt="The Abutment Slopes section showing a description with a help icon explaining the bench concept, and three fields in a horizontal row: Foundation/Abutment Profile dropdown (not yet selected), and two disabled numeric input fields for Right Abutment Slope (deg) and Left Abutment Slope (deg), each with a help icon next to the label."
  caption="Abutment Slopes section in its initial state."
/>

The section contains the following fields:

- **Foundation/Abutment Profile** — Indicates whether the abutment geometry is known or unknown. Options are "Known" or "Unknown." When set to "Unknown," the slope angle fields are disabled.
- **Right Abutment Slope (deg)** — The slope of the right abutment when looking downstream, measured in degrees from horizontal (0° = flat, 90° = vertical). A help icon next to the label explains the measurement convention. Enabled only when Foundation/Abutment Profile is set to "Known."
- **Left Abutment Slope (deg)** — The slope of the left abutment when looking downstream. Same measurement convention as the right abutment. A help icon next to the label explains the measurement convention. Enabled only when Foundation/Abutment Profile is set to "Known."

If there is a bench (a relatively flat step or terrace in the abutment slope), the user enters the angle of the steeper slope above the bench. A help icon in the section description explains this concept.

## Auto-Save Behavior

All inputs on the Dam Information page save automatically as the user makes changes. There is no Save button. Standard input fields (text boxes, dropdowns, numeric fields) save after a brief delay following the last keystroke. Text areas save when the user clicks out of the field.

A saving indicator appears briefly in the interface when a save operation is in progress. If the user navigates away from the page before saving completes, the DST preserves the most recently saved state.
