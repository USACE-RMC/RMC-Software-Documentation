# Documents

The Documents module manages file attachments associated with screenings. Users can upload supporting documents (PDFs, images, spreadsheets, etc.) to provide context and evidence for their screening assessments.

All tables in this chapter are planned and not yet implemented in the database.

## Storage Strategy

The storage strategy for document attachments has not been finalized. Two approaches are under consideration:

1. **File system storage** — Files stored on the server's file system with paths tracked in the database. Simpler to implement but requires file system management and backup coordination.

2. **Blob storage** — Files stored in cloud blob storage (e.g., Azure Blob Storage, S3) with references in the database. Better scalability and built-in redundancy but adds external service dependency.

The `StoragePath` column is designed to work with either approach — it can hold a file system path or a blob storage URI.
