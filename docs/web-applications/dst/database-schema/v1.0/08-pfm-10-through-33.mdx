# PFM-10 through PFM-33

PFM-10 through PFM-33 cover additional failure modes including sand boils (internal erosion), various specialized failure mechanisms, and user-defined failure modes. These modules follow the same **four-table blueprint** described in the previous chapter.

All tables in this chapter are planned except PFM-10 which has partial frontend support.

## PFM Index

| PFM | Name | Status |
|-----|------|--------|
| PFM-10 | Internal Erosion - Sand Boils | Partial (frontend only) |
| PFM-11 through PFM-32 | Various (names TBD) | Planned |
| PFM-33 | User-Defined Failure Mode | Planned |

## Table Count Summary

Each of the 24 PFMs in this range will produce 4 tables, for a total of **96 planned tables**. Combined with PFMs 1-9, the complete PFM module set will contain **132 tables** (4 implemented for PFM-01, 128 planned).

## Four-Table Pattern Recap

All PFMs follow the same structural pattern:

```
Pfm{NN} (1) ──→ (0..1) Pfm{NN}_DstSrp       [FK, UNIQUE, CASCADE DELETE]
Pfm{NN} (1) ──→ (0..1) Pfm{NN}_UserSrp      [FK, UNIQUE, CASCADE DELETE]
Pfm{NN} (1) ──→ (0..1) Pfm{NN}_ActiveSrp    [FK, UNIQUE, CASCADE DELETE]
```

The SRP child table structures are identical across all PFMs. Only the main `Pfm{NN}` entity table has PFM-specific columns for the failure mode's unique input parameters.
