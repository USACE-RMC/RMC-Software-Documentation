# Hazards

The Hazards module stores hydrologic and seismic hazard characterization data for each screening. Hazard data defines the loading conditions (flood frequencies, seismic ground motions) used as inputs to failure mode analyses.

All tables in this chapter are planned and not yet implemented in the database.

## Hazard Curve Storage

Hazard curves (stage-frequency, PGA-frequency) are stored as individual rows in child tables rather than JSONB columns. This normalized approach was chosen because:

- Hazard curve points need individual querying and filtering for interpolation
- The number of points is relatively small (typically 10-30)
- Individual points may need to be edited independently
- Relational storage enables straightforward validation constraints
