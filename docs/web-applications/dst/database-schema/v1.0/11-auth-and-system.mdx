# Auth & System

The Auth & System module covers authentication, user management, role-based access control (RBAC), session management, audit logging, and dashboard configuration. These tables support the administrative infrastructure of the DST application.

All tables in this chapter are planned and not yet implemented in the database.

**Tables in this chapter:**
[Users](#Users) · [Roles](#Roles) · [UserRoles](#UserRoles) · [Sessions](#Sessions) · [AuditLog](#AuditLog) · [DashboardConfig](#DashboardConfig)

## RBAC Design

The DST RBAC model uses a simple role-based approach:

| Role | Permissions |
|------|-------------|
| **Admin** | Full access — manage users, roles, all screenings |
| **Reviewer** | View and comment on screenings, approve/reject |
| **Analyst** | Create and edit screenings, run analyses |
| **Viewer** | Read-only access to shared screenings |

Role assignments are stored in the `UserRoles` join table. A user can have multiple roles, and permissions are evaluated as the union of all assigned role permissions.
