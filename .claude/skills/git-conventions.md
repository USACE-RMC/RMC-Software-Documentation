# Git Conventions

Reference file for git-related skills.

## Commit Messages

- Start with action verbs: Add, Update, Fix, Remove, Refine, Rework, Merge
- Concise and descriptive (one line preferred, multi-line body for complex changes)
- Do NOT include "Co-Authored-By" or any AI attribution
- No conventional commits prefix (feat:, fix:) — plain English
- Focus on the "why" more than the "what"

### Recent examples from this project

```
Overhaul color scheme from teal to steel blue with tokenized surfaces
Redesign homepage with teal gradient product tiles and frosted glass icons
Add figure reference popovers, nowrap references, and CSS refinements
Resolve LifeSim User's Guide v1.0 Level 3 QC findings
```

## Branch Naming

- Feature branches: `feature/{descriptive-name}` (e.g., `feature/site-upgrades`)
- Bug fixes: `fix/{descriptive-name}`
- Documentation: `docs/{descriptive-name}`

## Pull Requests

- Title: Short, action-verb-first, under 70 characters
- Body format:
  ```
  ## Summary
  - Bullet point 1
  - Bullet point 2
  ```
- **No "Test plan" section** — keep the body to Summary only
- **No AI attribution** — do not mention Claude, AI, or include "Generated with" footers
- Base branch: `main`
- Always create from a feature/fix branch, never from main

## Safety Rules

- Never force push to `main` or `master`
- Never use `--no-verify` to skip hooks
- Prefer new commits over `--amend` (amend only when user explicitly asks)
- Never auto-commit without user confirmation
- Stage specific files by name, never `git add -A` or `git add .`
- Never commit `.env`, credentials, or secrets

## Auto-Generated Files

These files are regenerated by build scripts and should generally not be committed manually:
- `sidebars.js`
- `src/reportIdMap.js`
- `static/counters/` contents
- `static/versions/` contents
- `src/data/` contents
